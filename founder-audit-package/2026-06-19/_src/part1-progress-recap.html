What TalisMan is

A local-first, approval-gated AI orchestration system: it coordinates Claude Code and Codex CLI as worker agents to carry projects through a governed "spiral" (discovery → plan → review → …), pausing at human-approval gates and enforcing cost and security boundaries. The design principle throughout is gated autonomy — it works unattended until a decision, a budget event, or an incident needs a human.

How it was built — the meta-story

TalisMan was built by its own governed method (we dogfooded). Every change shipped as one slice = one branch = one PR, and had to pass three independent gates before merging:

GateWhat it isWhy it matters
1 · Checks (CI)ruff, ruff-format, mypy --strict, pytest, import-linter — in CI on every PRMachine-objective; never skipped
2 · Cross-familyThe opposite AI family reviews every slice (Claude leads → Codex reviews, and vice-versa); saved as a YAML artifactIndependent eyes catch what tests can't
3 · Human gateYou merge the PR (branch protection on main)Final say stays human

The standout moment — the governance caught real bugs

Cross-family review wasn't ceremony. In S11.01 Codex blocked a genuine cap-bypass bug that had passed every automated check. And in S15.01 — the release decision itself — Codex blocked the lead (Claude) for over-grading its own work, forcing an honest re-grade. A system that catches its own optimism is the whole point.
